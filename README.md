3. Run the app:
   `npm run dev`

## 🚀 **CORE FEATURES & SYSTEM CAPABILITIES**

**THE PLATFORM OPERATES ACROSS THREE DEDICATED USER ROLES TO MAINTAIN STRICT DATA BOUNDARIES AND CLEAR WORKFLOWS:**

### **1. PATIENT PORTAL**
*   **PROFILE MANAGEMENT:** **SECURE REGISTRATION AND MEDICAL HISTORY LOGS.**
*   **APPOINTMENT BOOKING:** **REAL-TIME SCHEDULING WITH SPECIALIZED DOCTORS BASED ON AVAILABILITY.**
*   **MEDICAL RECORDS ACCESS:** **DIGITAL VIEWING OF PRESCRIPTIONS, LABORATORY SUMMARIES, AND DISCHARGE DETAILS.**

### **2. DOCTOR DASHBOARD**
*   **CONSULTATION QUEUE:** **ORGANIZED INTERFACE LISTING UPCOMING PATIENT VISITS.**
*   **E-PRESCRIPTIONS:** **DIRECT DIGITAL TOOL TO ADD DIAGNOSTIC NOTES, PRESCRIBE MEDICATIONS, AND ORDER FOLLOW-UP TESTS.**
*   **AVAILABILITY PLANNER:** **FLEXIBLE SCHEDULER TO MANAGE SHIFTS AND LEAVES.**

### **3. ADMINISTRATIVE CONSOLE**
*   **RESOURCE ALLOCATION:** **LIVE TRACKING OF HOSPITAL ROOMS, ICU BEDS, AND SPECIALIZED MEDICAL INVENTORY.**
*   **BILLING & INVOICING:** **AUTOMATED CALCULATION OF CONSULTATION FEES, ROOM CHARGES, AND PHARMACY TALLIES.**
*   **USER MANAGEMENT:** **CENTRALIZED SYSTEM TO AUTHORIZE NEW MEDICAL STAFF AND MANAGE DATABASE PERMISSIONS.**

## 🤖 **PROPOSED AI IMPLEMENTATION & SMART FEATURES**

**TO EVOLVE THIS STANDARD MANAGEMENT PLATFORM INTO A NEXT-GENERATION HEALTHCARE SOLUTION, THE FOLLOWING AI MODULES ARE ARCHITECTED TO ATTACH DIRECTLY TO THE SYSTEM BACKEND:**

### 🧠 **1. SMART PATIENT SYMPTOM TRIAGE (NLP & CLASSIFICATION)**
*   **HOW IT WORKS:** **WHEN A PATIENT ATTEMPTS TO BOOK AN APPOINTMENT, A TEXT-BASED INPUT CAPTURES THEIR PRIMARY SYMPTOMS. A NATURAL LANGUAGE PROCESSING (NLP) CLASSIFICATION MODEL MATCHES THE TEXT TO PROBABLE MEDICAL CATEGORIES.**
*   **IMPACT:** **AUTOMATICALLY RECOMMENDS THE CORRECT DEPARTMENT (E.G., CARDIOLOGY, NEUROLOGY) AND PRIORITIZES BOOKING URGENCY (TRIAGE LEVEL 1 TO 5) BASED ON SEVERITY.**

### 📊 **2. PATIENT LENGTH OF STAY (LOS) PREDICTOR (MACHINE LEARNING)**
*   **HOW IT WORKS:** **A REGRESSION MODEL BUILT ON HISTORICAL DATA (ANALYZING PARAMETERS LIKE AGE, INITIAL DIAGNOSIS, COMORBIDITIES, AND ADMISSION VITALS).**
*   **IMPACT:** **PREDICTS THE NUMBER OF DAYS A PATIENT WILL LIKELY OCCUPY A BED UPON ADMISSION, ALLOWING ADMINISTRATORS TO OPTIMIZE HOSPITAL BED CAPACITY AND STAFFING RATIOS.**

### 📝 **3. AI MEDICAL SUMMARY & SMART PRESCRIPTION ANALYZER (LLM INTEGRATION)**
*   **HOW IT WORKS:** **INTEGRATES A SECURE GENERATIVE AI API (LIKE GOOGLE GEMINI OR OPENAI) WITHIN THE DOCTOR DASHBOARD.**
*   **IMPACT:** **SUMMARIZES DENSE MULTI-PAGE PATIENT HISTORIES INTO BRIEF CLINICAL BULLET POINTS BEFORE A CONSULTATION, AND CROSS-REFERENCES NEW E-PRESCRIPTIONS AGAINST KNOWN PATIENT ALLERGIES OR DRUG INTERACTIONS.**

### **RUNNING LOCALLY**
1.  **CLONE THE REPOSITORY:**
    ```bash
    git clone [https://github.com/prem1382/Hospital-management-system.git](https://github.com/prem1382/Hospital-management-system.git)
    cd Hospital-management-system
    ```
2.  **CONFIGURE YOUR LOCAL ENVIRONMENT VARIABLES OR DATABASE CONFIGURATIONS INSIDE THE PRIMARY PROPERTIES/CONFIGURATION FILES.**
3.  **INITIALIZE YOUR DATABASE SCHEMA USING THE PROVIDED SQL INITIALIZATION SCRIPTS.**
4.  **LAUNCH THE APPLICATION SERVER AND NAVIGATE TO `HTTP://LOCALHOST:PORT` IN YOUR BROWSER.**
